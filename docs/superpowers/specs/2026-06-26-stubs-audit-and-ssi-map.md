# Заглушки расширения — аудит, привязка к SSI, что строить

**Date:** 2026-06-26
**Зачем:** в UI много демо-заглушек (1:1 с эталоном — правильно), но без плана развития.
Здесь — что из них реально нужно, насколько реально сделать, и привязка к главной задаче:
**поднимать каждый из 4 столбов SSI**.

## Ключевая рамка: РЫЧАГИ vs ВИТРИНА

LinkedIn SSI = 4 столба. Каждый поднимается конкретным действием бота (**рычаг**).
Большинство заглушек — это **витрина** (показывает результат), а не рычаг. Витрина не
двигает SSI; миссия — рычаги.

| Столб SSI | Что поднимает (рычаг) | Статус |
|---|---|---|
| **insights** (Обмен инсайтами) | лайки + комменты в ленте | ✅ **готово** (engagement + comments) |
| **brand** (Проф. бренд) | публикация постов | 🟡 идеи→черновик готовы; **публикация = Layer 2 (todo)** |
| **people** (Поиск людей) | находить + коннектиться с рекрутёрами/ЦА | 🔴 **Smart Connect (todo)** |
| **relationships** (Построение связей) | коннекты + поддержание сети | 🔴 **Smart Connect (todo)** |

**Вывод:** дорожная карта рычагов = engagement ✅ → **публикация контента (Layer 2)** →
**Smart Connect**. Всё остальное (Входящие, Эффект, Цель недели) — витрина.

## Разбор заглушек

### 1. «Цель недели» (Dash, баннер-хардкод)
- **Что должно:** взять 4 балла столбов, найти самый слабый, сказать «фокус сюда» + каким модулем поднимать.
- **Нужна ли LLM?** **НЕТ.** Маппинг «слабый столб → рычаг» фиксированный (insights→вовлечённость, brand→контент, people/relationships→Smart Connect). Это чистая детерминированная функция от уже имеющихся баллов (`SsiSnapshot.pillars`). LLM ничего не добавляет, только цена и недетерминизм.
- **Реально:** ✅ дёшево, ~маленький инкремент, pure-логика + тест.
- **Нужно:** ДА — прямо операционализирует «подними слабый столб». Направляет юзера.
- **Решение:** **СТРОИТЬ** (rule-based). В спеку.

### 2. SSI «▲ +14 за 14 дней» (Dash, хардкод-дельта)
- **Что должно:** реальная динамика SSI за период.
- **Реально:** ✅ дёшево — хранить дневные снапшоты SSI, считать дельту.
- **Нужно:** ДА — реальный прогресс мотивирует; это «доказательство работы».
- **Решение:** **СТРОИТЬ** (SSI-history store + дельта). В спеку.

### 3. «Входящие» (вкладка Inbox, `DEMO_LEADS`) — твой вопрос «реально ли + как отличить рекрутёра»
- **Что должно:** показать, кто написал тебе / смотрел профиль (главная метрика продукта — «сколько написали ТЕБЕ»).
- **Реально:**
  - *Кто смотрел профиль* — LinkedIn прячет список за Premium (free видит ~последних, часто размыто). На free-аккаунте данные **урезаны LinkedIn**. ⚠️ низко-средне.
  - *Кто написал* — парсинг твоего **личного инбокса** LinkedIn. Технически можно, но это **чтение приватных переписок** — инвазивно, privacy-чувствительно.
  - *Рекрутёр vs разраб vs спамер* — по **хедлайну отправителя** (recruiter/talent/sourcing/hiring) тем же `RelevanceScorer`. Рекрутёра отличить дёшево и норм. **Спам — ненадёжно** без LLM-разбора текста писем (а гонять личные DM через LLM = privacy + цена).
- **Нужно?** Это **витрина результата, не рычаг** — SSI не двигает, только показывает. Плюс самая сложная + privacy-тяжёлая + урезана LinkedIn.
- **Решение:** **ОТЛОЖИТЬ/НЕ ФЕЙКОВАТЬ.** Не строить чтение DM сейчас. Возможный лёгкий V2: парсить только «кто смотрел профиль» + помечать рекрутёров по хедлайну (без чтения переписок). Но это после рычагов. Сейчас — честная заглушка («появится позже»), а не фейковые лиды.

### 4. «Эффект на этой неделе» (Dash: 7 входящих / ×3.2 просмотров / 41% accept)
- **входящих** → зависит от «Входящие» (см. 3, отложено).
- **просмотров профиля** → как 3, урезано LinkedIn на free.
- **accept rate коннектов** → нужен Smart Connect (считать sent vs accepted). Появится с ним.
- **Решение:** не фейковать числа. Показывать реальное-или-«—». Большинство приедет вместе со Smart Connect / Inbox-V2.

### 5. Карточки «Модули»: smart_connect / auto_apply
- **smart_connect** («Скоро») — рычаг для people+relationships. **Нужно**, реально, но требует живого DOM-recon кнопок Connect/Add note (твоя сессия). Уже намечено фазой.
- **auto_apply** («Появится после ядра V1») — авто-отклики на вакансии. **SSI НЕ поднимает** (подача заявок ≠ SSI). Это территория Job Radar. **Решение: убрать из scope SSI** (оставить честную заглушку «вне ядра / через Job Radar»).

## Что добавить в спеку / порядок

**Строить сейчас (дёшево, двигает SSI или мотивацию, без твоей сессии):**
1. **«Цель недели»** — слабый столб → рекомендованный модуль (rule-based, pure, TDD).
2. **SSI-тренд** — история снапшотов + реальная дельта «за N дней».

**Следующие рычаги (по приоритету миссии):**
3. **Контент Layer 2** — публикация черновика (поднимает brand). Нужен composer DOM-recon (твоя сессия).
4. **Smart Connect** — коннекты рекрутёрам + Note (поднимает people+relationships). Нужен DOM-recon.

**Отложить/убрать (витрина, сложно/инвазивно/не про SSI):**
5. «Входящие» (чтение DM) — privacy + урезано LinkedIn + не рычаг. Не фейковать; честная заглушка.
6. «Эффект недели» числа — не фейковать; реальное-или-«—», приедет со Smart Connect/Inbox-V2.
7. **auto_apply** — вне SSI-миссии; убрать из ядра.

**Принцип:** убрать фейковые числа из дашборда — честный «—»/«позже» лучше выдуманных
метрик (фейк подрывает доверие к реальным).
